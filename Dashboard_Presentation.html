Executive Summary

From purchase records to BIR-ready compliance files

A browser-based dashboard that consolidates purchase transactions, input VAT, and expanded withholding tax for a selected fiscal year and month, lets finance verify every transaction, and generates the official BIR outputs — Summary List of Purchases, QAP 1601-EQ, and the subsidiary books — in both Excel and DAT formats. It optionally syncs to a shared cloud database.

Calculations

How VAT, EWT, and balances are computed

VAT

VAT = vatable base × VAT Category rate.
S / G / I / CG = 12%; SNQ / GNQ / uncoded = 0%. The category also splits amounts into goods, services, capital goods, and exempt for the SLP.

EWT

EWT = amount × ATC Code rate.
Rates come from the ATC Master. A missing rate is flagged for review.

Balance check

Computed Purchase VAT and EWT per CV are compared with the uploaded ledger balances; differences within a small rounding tolerance are "Balanced", otherwise "Review balances".

Compliance score

Each CV group rolls up its line statuses into an overall status and score for the Compliance Summary.

Verification

Six transaction statuses

Each purchase line carries a verification status that drives the Compliance Summary and the exports.

Compliant Without Invoice Non-Compliant Unreviewed Journal Entry Adjusting Entry
  • Journal Entry — intentional posting; valid without invoice, VAT category, ATC, or TIN (not flagged incomplete).
  • Adjusting Entry — same field exemptions, and excluded from all BIR Compliance Exports.

SOP Summary

The monthly routine

  • Select the fiscal year & month.
  • Import the QuickBooks Purchase, VAT, and EWT working papers (auto-detected).
  • Verify each CV line; set the verification status.
  • Reconcile VAT & EWT against uploaded balances; resolve "Review balances".
  • Maintain Supplier Master / ATC / VAT Categories (BIR character rule enforced).
  • Review the Compliance Summary; clear outstanding items.
  • Export the required BIR reports for the single selected month.
  • Archive the source files; keep cloud sync current.

Limitations & Next Steps

Honest status

  • Static front-end: when cloud sync is used, the Supabase URL & publishable key live in a browser config file and can't be fully hidden without a backend.
  • Company filer profile (TIN/name/address for BIR headers) is a code constant today. Planned in-app Settings screen.
  • Single shared database by design; not multi-tenant.
  • BIR formats should be spot-checked against current RDO requirements on first use.
  • Next: verify a real SLP/QAP export, optionally move sync to the company's own Supabase, then run the monthly SOP cadence.
